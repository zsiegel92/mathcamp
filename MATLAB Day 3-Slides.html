Approximation of the value of 𝜋π.¶

Steps to approximate:

  1. Uniformly scatter n points over a unit square [0,1]×[0,1].
  2. For each point, determine whether it lies inside the unit circle.
  3. The percentage of points inside the unit circle is an estimate of 𝜋/4π/4. Multiply the percentage by 4 to estimate 𝜋π.
  • Create a function that gives you a single estimate of 𝜋π for each n.
  • How does the approximation error change, as n changes?
In [48]:
iter=100000; % the experiment event number
radius=1; % the circle radius
inside=0; %  sucessful event number 
for i=1:iter
  x=rand();
  y=rand();
  if ((x^2+y^2)<=radius^2)
    inside = inside+1;
  end
end

pi_sim=4*inside/iter

    3.1452

Digression: Numerical differentiation¶

To check if our solution is a maximum, we can check the second derivative of our function. We can do that by considering the numerical derivative of our function evaluated at our solution.

The derivative of a function is:

𝑓(𝑥)=lim𝜖→0𝑓(𝑥+𝜖)−𝑓(𝑥)𝜖.f(x)=limϵ→0f(x+ϵ)−f(x)ϵ.

Thus, a natural approximation is the “forward differencing” scheme:

𝑓(𝑥)=𝑓(𝑥+𝜖)−𝑓(𝑥)𝜖f(x)=f(x+ϵ)−f(x)ϵ

for 𝜖ϵ small enough. Similarly, we can evaluate multivariable derivatives. Using the taylor rule, we can calculate the approximation error for our approach, and as you can imagine there are much better ways to approximate a derivative numerically.

To avoid approximating derivatives numerically, we can use the Symbolic Math Toolbox. However, you need to remember that there is important performance penalty of using the symbolic MATLAB language. Below is an example using the toolbox:

>> syms f(x)
>> f(x) = prod*x.^(1-ttheta)*capital^(ttheta) ;
>> df = diff(f,x)

df(x) =

30561625416561825/(562949953421312*x^(1/3))

>> dff = diff(df,x)

dff(x) =


-10187208472187275/(562949953421312*x^(4/3))

To evaluate the symbolic expression, we use the command subs:

subs(dff(fsolve(calculateMCapProf,200)))

ans =

   -0.0208

As you can see the expression is horrendous, but at least we can evalueate the 2nd order condition easily

For this example, we will use a methor of numerical differentiation called Richardson’s extrapolation. Matlab may be used as follows to find the derivative using Richardson extrapolation. Note that matrices start at (1,1), so we must make an adjustment. We assume a function y = f(x) is defined. Then, we can approximate the derivative by:

𝑓′(𝑥)≈−𝑓(𝑥+2ℎ)+8𝑓(𝑥+ℎ)−8𝑓(𝑥−ℎ)+𝑓(𝑥−2ℎ)12ℎf′(x)≈−f(x+2h)+8f(x+h)−8f(x−h)+f(x−2h)12h

Look at pages 8-9 for the proof.

Description¶

Let time be denoted as 𝑡=0,1,2,...,𝑇t=0,1,2,...,T, where 𝑇T is a finite number. Given an initial level of capital 𝑘⎯⎯⎯0k¯0, the benevolent social planner chooses sequences of capital {𝑘𝑡}𝑇𝑡=1{kt}t=1T and consumption {𝑐𝑡}𝑇𝑡=0{ct}t=0T to maximize the discounted present value of a representative agent's utility subject to a sequence of resource constraints:

max{𝑘𝑡}𝑇𝑡=1,{𝑐𝑡}𝑇𝑡=0∑𝑡=0𝑇𝛽𝑡log(𝑐𝑡)max{kt}t=1T,{ct}t=0T∑t=0Tβtlog⁡(ct)

subject to

𝑘𝑡+1=𝐴𝑘𝛼𝑡+(1−𝛿)𝑘𝑡−𝑐𝑡 for 𝑡=0,1,2,3...,𝑇kt+1=Aktα+(1−δ)kt−ct for t=0,1,2,3...,T

and 𝑘⎯⎯⎯𝑇+1k¯T+1, the terminal value of capital, is also given.

The first order necessary conditions for this problem reduce to the following set of equations:

0=[𝐴𝑘𝛼𝑡+1+(1−𝛿)𝑘𝑡+1−𝑘𝑡+2]−𝛽[𝐴𝑘𝛼𝑡+(1−𝛿)𝑘𝑡−𝑘𝑡+1](𝛼𝐴𝑘𝛼−1𝑡+1+(1−𝛿)),∀𝑡=0,1,2...,𝑇0=[Akt+1α+(1−δ)kt+1−kt+2]−β[Aktα+(1−δ)kt−kt+1](αAkt+1α−1+(1−δ)),∀t=0,1,2...,T

Analysis¶

We can think of these first order conditions as a list of second order difference equations of the form:

0=𝑔(𝑘⎯⎯⎯0,𝑘1,𝑘2)0=g(k¯0,k1,k2)
0=𝑔(𝑘𝑡,𝑘𝑘+1,𝑘𝑘+2),∀𝑡=0,1,2...,𝑇0=g(kt,kk+1,kk+2),∀t=0,1,2...,T
0=𝑔(𝑘𝑇−1,𝑘𝑇,𝑘⎯⎯⎯𝑇+1)0=g(kT−1,kT,k¯T+1)

This is a system of 𝑇T equations in 𝑇T unknowns {𝑘𝑡}𝑇𝑡=1{kt}t=1T. Denote {𝑘∗𝑡}𝑇𝑡=1{kt∗}t=1T as the solution to this system of equations.
